## Overview

Alumni by Better's interface is built on **Liquid Glass** — a design language where translucent cards with frosted blur effects float over subtle color gradients, creating depth without heaviness. The system pairs this modern glass aesthetic with the warmth of Cormorant Garamond serif headlines, producing an interface that feels simultaneously contemporary and editorial.

Every surface serves a pedagogical purpose. Cards contain exercises, vocabulary, or speech trainers. Buttons are always solid-filled (never ghost-only). Navy blue (#003080) anchors every interactive element. The background is never flat white — always a soft gradient that makes the glass blur visible. The result is a learning platform that feels premium, calm, and deeply structured.

**Key Characteristics:**
- Liquid Glass translucency on all card surfaces — backdrop-filter blur with semi-transparent backgrounds.
- Warm gradient background (never flat white) — the gradient is essential for the glass effect to be visible.
- Two-font system: Cormorant Garamond (serif, display) for headlines and phrase display; Inter/system-ui (sans-serif) for body, exercises, and UI elements.
- Single primary color: Navy #003080 carries every interactive element — buttons, links, badges, progress bars.
- Three action button colors: Navy (Listen), Green (Check/Verify), Red (Record). All with glass treatment (translucent + blur + inset light reflection).
- Inset shadow light reflection on card tops — `inset 0 1px 0 rgba(255,255,255,0.4)` simulates light hitting frosted glass.
- Exercise cards use `data-exercise` attributes and auto-render via exercises.js — zero inline JavaScript.
- Vocabulary cards are always vertical list (flex-direction: column), never multi-column grid.
- Lesson cards are expandable (collapsed by default) with smooth fadeIn animation.
- Zero emojis — only Lucide SVG icons (stroke-based, 18px).
- WCAG AAA contrast (7:1) on header hero; WCAG AA (4.5:1) minimum everywhere else.
- Mobile-first responsive with 44x44px touch targets.

## Colors

### Brand & Interactive
- **Navy Primary** (`{colors.primary}` — #003080): The single interactive color. All Listen buttons, links, progress bars, badges, active states. This is Alumni's "click me" signal.
- **Navy Hover** (`{colors.primary-hover}` — #002060): Darkened state for button hover.
- **Navy Light** (`{colors.primary-light}` — rgba(0,48,128,0.08)): Subtle background for "Listen All" button and navy-tinted surfaces.
- **Navy Translucent** (`{colors.primary-translucent}` — rgba(0,48,128,0.85)): The glass treatment for Listen buttons — 85% opacity lets the blur show through.
- **Accent Red** (`{colors.accent}` — #d70c0c): Reserved for Alumni brand accent moments. Not used on buttons (those use `{colors.danger}`).

### Action Buttons
- **Success Green** (`{colors.success}` — #16a34a): Check/Verify buttons and correct feedback.
- **Success Translucent** (`{colors.success-translucent}` — rgba(22,163,74,0.85)): Glass treatment for Check buttons.
- **Danger Red** (`{colors.danger}` — #dc2626): Record buttons and error feedback.
- **Danger Translucent** (`{colors.danger-translucent}` — rgba(220,38,38,0.85)): Glass treatment for Record buttons.
- **Warning Amber** (`{colors.warning}` — #b45309): Hints, tips, media-tip text.
- **Amber Solid** (`{colors.amber}` — #f59e0b): Stop button background.

### Surfaces
- **Canvas** (`{colors.canvas}` — #f5f5f0): Base page color — but NEVER used as flat `background: #f5f5f0`. Always as part of the gradient.
- **Canvas Gradient**: `linear-gradient(135deg, #e8eaf0 0%, #f5f5f0 50%, #e0e8f0 100%)` — the essential gradient that makes Liquid Glass work. Without it, the blur effect is invisible.
- **Card Warm** (`{colors.card-warm}` — #fafaf7): Solid fallback for speech-card and think-card backgrounds when glass isn't needed.
- **Card White** (`{colors.card-white}` — #ffffff): Clean white for exercise sections and quiz options.

| `{typography.badge}` | 11px | 700 | 1.0 | Badge labels (uppercase, tracked) |
| `{typography.lesson-number}` | 12px | 600 | 1.0 | "AULA 01 — PRE-CLASS" (uppercase, wide tracking) |
| `{typography.button}` | 14px | 600 | 1.0 | Button labels |
| `{typography.subtitle}` | 16px | 400 | 1.5 | Header hero subtitle |
| `{typography.info-pill}` | 14px | 400 | 1.5 | Header info pills (uppercase) |

### Principles
- **Cormorant Garamond for display only.** Never use it for body text, buttons, badges, or microcopy. It's reserved for moments that need editorial weight: names, titles, phrases.
- **Body at 15px, not 14 or 16.** Alumni runs paragraph text at 15px — large enough for comfortable reading, compact enough for exercise-dense layouts.
- **Weight 700 for hero only.** Lesson titles and section headers use 600. Only the header hero name uses 700.
- **Line-height context-specific.** Display sizes use 1.1-1.3 (tight). Body uses 1.6 (comfortable). Badges use 1.0 (single line).
- **Letter-spacing for structure.** Badges track at 1.2px, lesson numbers at 3px. All other text uses default (0) tracking.

## Layout

### Spacing System
- **Base unit:** 8px (with 4px sub-base for tight adjustments).
- **Tokens:** `{spacing.xxs}` 4px · `{spacing.xs}` 8px · `{spacing.sm}` 12px · `{spacing.md}` 16px · `{spacing.lg}` 24px · `{spacing.xl}` 32px · `{spacing.xxl}` 48px · `{spacing.section}` 64px.
- **Card padding:** 16-19px for vocab/speech cards, 24px for exercise sections, 32-40px for lesson body.
- **Gap between cards:** 10px (vocab), 8px (match/order items), 13px (speech cards), 19px (exercise sections), 24px (lesson cards).

### Grid & Container
- **Max content width:** 900px for lesson content, 100% for header hero.
- **Vocabulary cards:** Single-column flex (`flex-direction: column`). NEVER multi-column grid.
- **Media cards:** CSS grid with `auto-fill, minmax(300px, 1fr)` — responsive columns.
- **Match/Order grids:** Single-column flex with 8px gap.
- **Quiz options:** Single-column flex with 6px gap.

### Whitespace Philosophy
Every exercise section breathes. Cards never touch — minimum 10px gap. Section titles have clear separation from content below. Microcopy sits below titles with a subtle left border (3px #e5e5e0) as a quiet visual anchor. The lesson body has generous padding (32-40px) to prevent cramped exercise layouts.

## Elevation & Depth

| Level | Treatment | Use |
|---|---|---|
| Flat | No shadow | Cards in resting state, exercise items |
| Hover lift | `box-shadow: 0 4px 20px rgba(0,0,0,0.06)` | Lesson card hover |
| Glass elevation | `box-shadow: 0 8px 32px rgba(0,0,0,0.08), inset 0 1px 0 rgba(255,255,255,0.4)` | Lesson card hover (full glass) |
| Button glass | `box-shadow: 0 4px 16px rgba(0,0,0,0.1), inset 0 1px 0 rgba(255,255,255,0.3)` | Listen/Check/Record buttons |
| Button hover | `box-shadow: 0 6px 20px rgba(0,48,128,0.25)` | Button hover state |
| Header passport | `backdrop-filter: blur(20px)` + translucent bg | Progress bar container in header |

**Glass philosophy.** Elevation comes from translucency and blur, not from drop shadows. The `inset 0 1px 0 rgba(255,255,255,0.4)` on top of glass cards simulates light hitting frosted glass from above — this is the signature Alumni depth cue.

### Backdrop Filter Usage
- **Lesson cards:** `blur(12px) saturate(160%)`
- **Exercise sections:** `blur(8px)`
- **Vocab cards:** `blur(8px)`
- **Speech cards:** `blur(6px)`
- **Survival cards:** `blur(4px)` (subtle)
- **Buttons:** `blur(10px) saturate(150%)`
- **Header passport:** `blur(20px)`
- Always include `-webkit-backdrop-filter` for Safari compatibility.

## Do's and Don'ts

### Do
- Use `{colors.primary}` (Navy #003080) for every interactive element — buttons, links, progress bars, active badges.
- Set all headlines in Cormorant Garamond with weight 600 (700 only for header hero name).
- Apply the gradient background (`linear-gradient(135deg, #e8eaf0, #f5f5f0, #e0e8f0)`) to the body — it's essential for Liquid Glass to work.
- Include `-webkit-backdrop-filter` alongside `backdrop-filter` for Safari support.
- Use the `inset 0 1px 0 rgba(255,255,255,0.4)` shadow on glass cards for the light reflection effect.
- Make all buttons solid-filled with glass treatment (translucent + blur). Never ghost-only buttons for primary actions.
- Keep vocabulary cards in vertical single-column layout (flex-direction: column). Never use CSS grid with multiple columns.
- Use `data-exercise` attributes for all exercises — let exercises.js handle rendering.
- Include Alumni logo in header of every page (professor AND student).
- Maintain 44x44px minimum touch targets on all interactive elements.
- Use Lucide SVG icons (18px, stroke-width 2) — never emojis, Font Awesome, or Material Icons.

### Don't
- Don't use flat white (#fff or #f5f5f0) as body background — it kills the Liquid Glass effect.
- Don't apply dark theme or dark backgrounds outside the header hero overlay.
- Don't use Cormorant Garamond for body text, buttons, badges, or microcopy — it's display-only.
- Don't create multi-column grids for vocabulary cards — always single column.
- Don't write inline JavaScript for exercises — no onclick handlers, no setTimeout, no innerHTML. Use only `data-exercise` attributes.
- Don't make buttons with border-only styling (ghost buttons) for Listen/Check/Record — always solid/translucent fill.
- Don't use text lighter than #666677 for any readable content (WCAG 4.5:1 minimum).
- Don't skip the `backdrop-filter` on glass surfaces — without blur, the translucent backgrounds look like faded colors instead of glass.
- Don't use emojis anywhere in the material — Lucide SVG icons only.
- Don't set border-radius larger than 12px on lesson cards or 10px on content cards — the radius scale is deliberate.

### prefers-reduced-motion
- **Disable:** backdrop-filter blur, all keyframe animations (fadeIn, shake, pulse), transform transitions

- **Substitute:** Animated reveals with simple opacity fade (0.15s)

## Accessibility

- **Contrast:** WCAG AAA (7:1) on header hero. WCAG AA (4.5:1) minimum on all other text. Text muted floor: #666677.
- **Focus:** `outline: 2px solid #003080; outline-offset: 2px` on all focusable elements.
- **ARIA:** `aria-label` on icon-only buttons (Listen, Record, Stop, arrows). `role="button"` on clickable non-button elements. `tabindex="0"` on interactive divs.
- **Alt text:** All images (logo, lesson headers, media thumbnails).
- **Icons:** Lucide SVG inline only — 18x18px, stroke-width 2, currentColor inheritance. Never decorative icon fonts.
- **Motion:** Full prefers-reduced-motion support (see Animations section).

## Iteration Guide

1. Reference component YAML keys directly (`{component.lesson-card}`, `{component.vocab-card}`) when building UI.
2. Always apply the body gradient FIRST — then layer glass cards on top. Without the gradient, glass looks wrong.
3. Use `{colors.primary}` for interactive elements, `{colors.success}` for correct states, `{colors.danger}` for error states. Never mix.
4. Exercises use `data-exercise` attributes only. The exercises.js file handles ALL rendering, validation, feedback, and animation.
5. Vocabulary cards are ALWAYS vertical list. This is non-negotiable.
6. When in doubt about a component's glass treatment: more blur = more premium. Start at blur(8px) and adjust.
7. Test on Safari — always include `-webkit-backdrop-filter` alongside `backdrop-filter`.

## Known Gaps

- Liquid Glass requires a gradient background to be visible — pages with dynamic content that covers the gradient may need additional gradient layers.
- `backdrop-filter` performance on older devices may require fallback solid backgrounds via `@supports not (backdrop-filter: blur(1px))`.
- SF Pro equivalent: on non-Apple platforms, Inter from Google Fonts is the closest match for the body font. Cormorant Garamond is available on Google Fonts for all platforms.
- Print styles strip glass effects and show solid borders instead.
