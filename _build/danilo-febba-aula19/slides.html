Chapter 3

The Comparison Code

adjective + -er + than · more + adjective + than

Discovery

Listen and Notice

How do we compare two things? What comes after the green words?

Python is faster than Excel.

The new pipeline is more reliable than the old one.

The new server is better than the old server.

Cloud storage is more expensive than local storage.

PatternRuleExample
Short adjectiveadjective + -er + thanfast → faster than
Long adjectivemore + adjective + thanreliable → more reliable than
Irregularspecial form + thangood → better, bad → worse

We use a comparative adjective + than to compare two things. Short adjectives add -er (faster, bigger, easier). Long adjectives use more (more reliable, more efficient). A few are irregular: good → better, bad → worse, far → further. Never use -er and more together.

Common Mistake

Right vs Wrong

✗ WRONG

"Python is more fast than Excel."

✓ RIGHT

"Python is faster than Excel."

✗ WRONG

"This server is more faster."

✓ RIGHT

"This server is faster."

✗ WRONG

"It is more reliable that the old one."

✓ RIGHT

"It is more reliable than the old one."

✗ WRONG

"The new tool is more good."

✓ RIGHT

"The new tool is better."

Short adjectives take -er (faster). Long adjectives take more (more reliable). Never both. After the comparative, use than. good and bad are irregular: better, worse.
