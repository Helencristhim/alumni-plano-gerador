Chapter 4: The Code

The Language of Advice

Prefer, warn, and stay practical

Grammar Discovery

Notice the Modals

"I would rather use the open-source tool than the paid one."

"We had better decide this week, or we will miss the deadline."

"We are blocked anyway, so we may as well set up the tool now."

"I would rather not wait, and we had better start today."

ModalUseForm
would ratherpreferencewould rather + base verb (than...)
had betterstrong advice / warninghad better + base verb
may as wellno better option; why notmay as well + base verb

All three take the base verb with no "to": would rather use, had better decide, may as well set up.

Language Focus

Recommend with the Right Modal

Common Mistake

Wrong vs Right

✗ WRONG

"I would rather to use the free tool."

✓ RIGHT

"I would rather use the free tool."

✗ WRONG

"We would better decide this week."

✓ RIGHT

"We had better decide this week."

Use the base verb after these modals, with no "to". And it is had better, never "would better".

Dialogue

The Proposal with David

D
Ana, I read your recommendation. Just to be clear, which tool would you rather use, and why?
A
I would rather use the open-source tool than the paid one. The upside is that the whole team already knows it, so setup is fast.
D
That is a compelling reason. But some people want to wait until next sprint. Is that a problem?
A
Yes. We had better decide this week, or we will run out of time before the release. It is advisable to start early.
D
Fair. And what about the downside of the free tool? Are you hiding anything?
A
No, I am honest about it: less official support. But I still think it is the more prudent, worthwhile choice for us right now.
D
Makes sense. We are blocked on the API this afternoon anyway.
A
Exactly. So we may as well set up the tool now and test it while we wait. It is a viable, low-risk plan.
Did You Catch It?

About the Proposal

1. What is the first thing David wants Ana to clarify?
Which tool she would rather use, and why.
2. What concern does David raise about the timing?
That some people want to wait until next sprint to decide.
3. Why does David think setting up the tool now is a good idea?
Because the team is blocked on the API that afternoon anyway, so it is a good use of the dead time.

Whether her preference for the open-source tool is about cost or about speed of setup, and whether "decide this week" is a hard deadline or if there is room for one more day.

Listening 2

Priya's Recommendation Tip

A tip from your team lead. Sound first — no text.

0:000:00
1. What three things does Priya say a good recommendation should separate?
What you prefer (would rather), what you advise (had better), and what is just practical (may as well).
2. What does Priya say about the downside?
Be honest about it -- a recommendation that hides the risk is not compelling, just optimistic; state the upside and the downside and let the team decide.
