Stage 1.1: Vocabulary Cards

Vocabulary

Listen to each word and read the example. Tap Listen to hear it.

Sprint -- a short period of work, usually two weeks, to build one part of a product
"This sprint we built a new password button." (Ciclo de trabalho)
Demo -- a short, live showing of the work in action
"Then I show the demo of what already works." (Demonstração)
Backlog -- the list of tasks and ideas still waiting to be built
"A backlog is the list of tasks that are waiting." (Lista de pendências)
Outsider -- someone who is not part of the technical team
"I explain the work so an outsider can follow it." (Leigo)
Rephrase -- to say something again in simpler, clearer words
"If a word is too technical, I rephrase it." (Reformular)
Simplify -- to make something easier to understand
"My job is to simplify the update for the room." (Simplificar)
Analogy -- a comparison that explains a new idea with a familiar one
"I like a simple analogy: our app is like a house." (Analogia)
Summary -- a short description of the whole thing, the main points only
"A good summary gives the big picture first." (Resumo)
Feature -- a specific function a user can use in a product
"The photo feature is not ready yet." (Funcionalidade)
Bug -- a fault that makes part of the software work badly
"We found a bug that slowed the login screen." (Falha / erro)

Stage 1.3: Grammar in Context

GRAMMAR

Read the text and answer the questions.

Before the sprint review, Rafael writes a short note for the outsiders in the room. He knows most of them are not engineers, so he wants to simplify every line. "A backlog is the list that holds the tasks we have not built yet," he writes, and he rephrases the jargon. For the new feature, he says: "It is a button which lets a customer reset a password alone." He adds a small analogy — the app is like a house, and this sprint they built the front door. He is honest about the bug too: "We found a problem that slowed the login, so one part needs one more week." He ends with a one-line summary and an invitation to the live demo, because anyone who can see the work will trust it.

1. How does Rafael define a backlog for the stakeholders?
A As a finished feature ready to launch.
B As the list that holds the tasks they have not built yet.
C As a bug in the login screen.
2. Why does Rafael add the house analogy?
A To make a technical idea easy to picture.
B To show the team built a real house.
C To hide the bug from the stakeholders.
3. Why does he end with a live demo?
A Because slides are always better than a demo.
B Because the feature is fully finished.
C Because anyone who can see the work will trust it.

Stage 1.4: Grammar Tip -- Defining Relative Clauses

GRAMMAR

Usamos that / which / who para DEFINIR de qual coisa ou pessoa falamos — ideal para explicar jargão em palavras simples. / We use that / which / who to DEFINE which thing or person we mean — perfect for explaining jargon in plain words.

Word / PalavraUse / UsoExample / Exemplo
thatcoisas ou pessoas (neutro, muito comum)a button that lets you reset
whichapenas coisas / only thingsthe list which holds the tasks
whoapenas pessoas / only peoplea person who cares
Structure / Estruturanoun + that / which / who + verba bug that slowed the login

Key point / Ponto-chave: nunca use what no lugar de that / which ("a button what lets you reset" está errado). E não repita o sujeito depois do pronome: "a person who he cares" está errado — o certo é "a person who cares".

Stage 4: Situational Quiz

Quiz

Choose the best response for each sprint review situation.

A non-technical director asks what your team built. The clearest answer is:
A "We built a button that lets customers reset their password."
B "We refactored the auth endpoint and cleared the backlog."
C "We did a lot of technical things this sprint."
A stakeholder does not know the word "backlog". You say:
A "A backlog is the list what holds the tasks."
B "A backlog is the list that holds the tasks still waiting."
C "A backlog is a person who waits for tasks."
A feature is not ready because of a bug. The most honest, simple update is:
A "Everything is perfect and fully done."
B "The photo part has a bug, so it needs one more week."
C "The mutex on the worker thread deadlocked again."
