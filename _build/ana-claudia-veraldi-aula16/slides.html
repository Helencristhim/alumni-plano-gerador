Chapter 4: The Code

The Language of Hindsight

Rewrite a past that did not happen

Grammar Discovery

Notice the Conditionals

"If we had tested it in staging, the outage would not have happened."

"If we had documented the near miss, we would not be here now."

"If our monitoring were better, we would have caught it sooner."

"If we had logged it, we would already know the fix."

ConditionalUseForm
Thirda past that did not happenif + past perfect, would have + past participle
Mixed (past → present)past cause, present resultif + past perfect, would + base verb
Mixed (present → past)present truth, past resultif + past simple, would have + past participle

The would form always goes in the result — never after "if".

Language Focus

Look Back with the Right Conditional

Common Mistake

Wrong vs Right

✗ WRONG

"If we would have tested it, the outage would not have happened."

✓ RIGHT

"If we had tested it, the outage would not have happened."

✗ WRONG

"If we had documented it, we would not been here now."

✓ RIGHT

"If we had documented it, we would not be here now."

Never put would after "if" — use the past perfect (had tested). And in a present result, use the base verb: would not be, never "would not been".

Dialogue

The Postmortem with David

D
Ana, I read your postmortem for the outage. Before we share it, can you tell me the single most important cause?
A
Sure. If we had tested the new login flow in staging, we would not have shipped the broken build. The outage was preventable.
D
Agreed. And you called the switched-off alert a contributing factor, not the root cause. Why the difference?
A
Because the missing check was the root cause. The alert was an oversight that made it worse, but it did not cause the bug by itself.
D
That is fair. There was also a near miss last sprint, right? Does that change anything now?
A
It does. If we had documented that near miss, we would not be here now. The same problem came back because nobody wrote it down.
D
Understood. And you kept the whole thing blameless. Was that on purpose?
A
Completely. A postmortem can backfire if it turns into blame. In hindsight, the honest takeaway is simple: test every login change.
