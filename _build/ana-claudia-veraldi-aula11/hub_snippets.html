If This, Then That -- First Conditional

Key words: condition, dependency, threshold, load, to overload, bottleneck, timeout, fallback, retry, rollout, to cascade, consequence. Stage 1.1: Vocabulary Cards

Vocabulary

Listen to each word and read the example. Tap Listen to hear it.

Condition -- a state that must be true for something else to happen
"If this condition is true, the test will fail."
Dependency -- something a task or feature relies on to work
"If a dependency is down, the feature will not load."
Threshold -- the level or point at which something starts to happen
"If the load passes the threshold, the server slows down."
Load -- the amount of traffic or work a system has to handle
"If the load is high, the app will respond slowly."
To overload -- to give a system more work than it can handle
"If too many users log in, they will overload the server."
Bottleneck -- a single slow point that holds up the whole system
"If the database is a bottleneck, every page will be slow."
Timeout -- an error when a system waits too long for a response
"If the server is slow, the request will end in a timeout."
Fallback -- a backup option the system uses if the main one fails
"If the main service fails, the fallback will take over."
Retry -- an automatic new attempt after something fails
"If the first attempt fails, the retry will run automatically."
Rollout -- the process of releasing a feature to users
"If the tests pass, the rollout will happen on Friday."
To cascade -- to spread from one failure to many in a chain
"If one service fails, the errors will cascade."
Consequence -- the result or effect that follows an action
"If we skip the test, the consequence will be a bug in production."

Stage 1.3: Grammar in Context

GRAMMAR

Read the text and answer the questions.

Before a rollout, Ana thinks in cause and effect. She looks at each condition and asks what will happen next. If the load is high on Monday morning, the app will slow down. If too many users arrive at once, they will overload the server. And if the database is a bottleneck, every page will be slow. She also checks the dependencies: if one service fails, the errors will cascade through the system. So the team adds a fallback. If the main service ends in a timeout, the app will switch to the backup. Notice the pattern: the condition uses the present simple ("if the load is high"), and the consequence uses "will" ("the app will slow down"). Never "will" after "if".

C "If the load is high, the app slow down."
2. In the first conditional, the verb after "if" is in the...
A present simple, with no "will".
B future, always with "will".
C past simple.
3. We use the first conditional to talk about...
A a real cause and its likely effect.
B an impossible, imaginary situation.
C something that finished in the past.

Stage 1.4: Grammar Tip -- First Conditional (Real Situations)

GRAMMAR

How to talk about a real cause and its likely effect in testing.

PartFormExample
If-clause (cause)If + present simpleIf the load is high,
Result (effect)will + base verbthe app will slow down.
Negativewill not / won'tIf we add a fallback, the users won't see an error.
Questionwill + subject...?If the load is high, will the app crash?
Key rulePresent simple after if (never "will" after if). You can swap the order: "The app will slow down if the load is high" (no comma when the if-clause is second).
