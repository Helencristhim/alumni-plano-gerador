How to Explain It in Sixty Seconds

Grammar Discovery

Listen and Notice

Which one translates? Which one compares? Which one makes you see it?

"We are replacing the database, which means the system does not stop on Mondays."

"The new system is cheaper to maintain than the old one."

"The legacy database is like a car from 2010."

"In plain terms, we lose four hours every week."

Grammar Rule

Translate, Compare, Illustrate

MoveStructureExample
Translate the technologytechnical fact + which means + business resultWe are replacing the database, which means the system does not stop.
Comparecheaper / faster / safer (+ than) -- never "more cheaper"The new system is cheaper to maintain than the old one.
IllustrateIt is like + something everybody knowsThe legacy database is like a car from 2010.

Start with the result, never with the technology. The executive is not buying a database -- she is buying four hours a week, a lower risk, and a bill she can defend. Say that first. The technology is one sentence, and it comes second.

Common Mistake

Watch Out

"The new system is more cheaper to maintain."
"The new system is cheaper to maintain."
"We chose it because is faster and more safe."
"We chose it because it is faster and safer."

Adjetivos curtos ganham -er (cheap → cheaper, safe → safer) -- nunca more + -er juntos. E toda oração em inglês precisa de sujeito: because it is faster.

Artefact

The One-Slide Answer

Steering Committee -- Database Replacement

Four Weeks. Here Is Why.

Owner: Diogo Leal

In plain terms: we spend four hours every week keeping a fifteen-year-old system alive

ABenefit: those four hours go back to client work, every week
BAnalogy: it is like a car from 2010 -- six cheap repairs a year
CRisk: the legacy database stores client data, and it is out of support
✓Cost of doing nothing: the same four hours a week, forever

One slide. Sixty seconds. Zero acronyms. If they want the architecture, they will ask.

Wrap-up

What I Learned

I open with money, time or risk -- never with the technology.
I can translate a technical fact with "which means".
I use comparatives correctly: cheaper, faster, safer.
I can explain a system with an analogy from real life.
I can say the cost of doing nothing.
