Lesson 04 -- Pre-class

Reading a Bug Report -- Listening for Detail

Reading a real bug report and catching every detail in a developer's voice message. Key words: severity, steps to reproduce, expected result, actual result, to crash, root cause, workaround, intermittent, error log, screenshot, environment, to escalate. Structure: past simple vs past continuous to describe what happened.

Stage 1.1: Vocabulary Cards

Vocabulary

Listen to each word and read the example. Tap Listen to hear it.

Severity -- how serious a bug is, from minor to critical
"The severity of this bug is critical."
Steps to reproduce -- the exact actions that make a bug appear again
"Under steps to reproduce, list every action."
Expected result -- what the software is supposed to do
"The expected result is a payment screen."
Actual result -- what the software really does instead
"The actual result is a blank screen."
To crash -- to stop working suddenly and completely
"The page crashed when I clicked Pay now."
Root cause -- the real, underlying reason for a problem
"Lucas is looking for the root cause."
Workaround -- a temporary fix that avoids a problem until it is solved
"We have a workaround: pay from the old page."
Intermittent -- happening only sometimes, not every time
"The bug is not intermittent; it happens every time."
Error log -- the record the system keeps of errors and events
"I attached a copy of the error log."
Screenshot -- a picture of the screen used as evidence
"I added a screenshot to the report."
Environment -- the setup where a bug appears, like browser and version
"The environment was Chrome 120 on staging."
To escalate -- to pass a serious issue to a higher level or team
"We had to escalate the critical bug."

Stage 1.3: Grammar in Context

GRAMMAR

Read the text and answer the questions.

Yesterday, Ana was testing the new checkout when she found a bug. She was adding items to the cart, and the total was updating correctly. Then she clicked Pay now, and the page crashed. She wrote a clear report. Under steps to reproduce, she listed every action. The expected result was a payment screen; the actual result was a blank page. She set the severity to critical, attached a screenshot and the error log, and noted the environment: Chrome 120 on staging. The bug was not intermittent; it happened every time. Lucas is still looking for the root cause, but there is a workaround: pay from the old page.

1. Why do we say "She was testing the checkout when she found a bug"?
A The testing was in progress (past continuous) when a short event (found) interrupted it.
B Both actions were long and happened at the same fixed time.
C It describes a habit she has every day.
2. "The page crashed when she clicked Pay now" uses the past simple because:
A both actions were long and in progress.
B both are short, completed events, one right after the other.
C it is a plan for the future.
3. Which sentence about the bug is correct?
A "I was test the checkout when it crashed."
B "I was testing the checkout when it crashed."
C "I testing the checkout when it was crashed."

Stage 1.4: Grammar Tip -- Past Simple vs Past Continuous

GRAMMAR

How to describe what happened when you found a bug.

UseFormExample
Action in progresswas / were + -ingI was testing the checkout.
Sudden, completed eventpast simpleThe page crashed.
Background + interruptionpast continuous + when + past simpleI was testing when it crashed.
Two background actionswhile + past continuousWhile the app was loading, I was watching the log.
Common mistakeDo not drop the -ing: NOT "I was test". Do not use the past continuous for a sudden event: NOT "the page was crashing when I clicked".
