Chapter 1: The Analysis Floor

Cause to Result

"The service crashed because the server was overloaded." (result, then cause)

"The server was overloaded, so the service crashed." (cause, then result)

"The capacity was low. As a result, the latency went up." (formal result)

"Because" points to the cause. "So" points to the result.

Chapter 3

The Link Rule

Because, since, so and as a result

Link Discovery

Listen and Notice

The service crashed because the server was overloaded.

The server was overloaded, so the service crashed.

Since the latency was high, requests timed out.

The capacity was low. As a result, the system slowed down.

Which word comes before the cause? Which comes before the result?

Link Rule

Cause and Result

ConnectorIntroducesExample
because / sinceThe cause (the reason)"...because the server was overloaded."
soThe result (informal)"...so the service crashed."
as a result / thereforeThe result (formal)"As a result, the latency went up."
cause first vs result first"X happened because Y" = "Y, so X." Same idea, opposite order.
Common Mistake

Wrong Link vs Right Link

✗ WRONG

"The server was overloaded because the service crashed." (reversed)

✓ RIGHT

"The service crashed because the server was overloaded."

✗ WRONG

"Because it was overloaded, so it crashed."

✓ RIGHT

"Because it was overloaded, it crashed." (because OR so, not both)

Put "because/since" before the cause and "so/as a result" before the result. Never use "because" and "so" together.
