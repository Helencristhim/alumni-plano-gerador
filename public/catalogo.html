O Modelo é o molde reutilizável — shell (layout/CSS/JS), os tipos de aula que ele monta e os gates que o validam. Todo aluno nasce de um modelo. O Perfil é a persona: um aluno de exemplo com seu Perfil 360, que escolhe um modelo e o preenche.

Modelo = o molde

A estrutura reutilizável. Não tem nome de gente. Bug de layout se corrige aqui e chega a todos os alunos via builder.

Ex.: Modelo Adulto · Modelo Kids

Perfil = a persona

Um aluno concreto com seu Perfil 360. Escolhe um modelo e traz o conteúdo dele (interesses, nível, objetivos).

Ex.: Helen Mendes (adulto) · Bento (kids)

Modelos

O molde que cada aluno herda. Os gates precisam passar em TODAS as aulas de qualquer modelo.

Os gates valem para TODAS as aulas de TODOS os modelos

Um modelo só é "pronto" quando cada aula passa em todos os gates dele. Modelo novo (ou tipo novo) entra primeiro no shell, com seus gates próprios — e sempre que um defeito escapar, cria-se um gate novo pra ele nunca mais voltar. Correção de layout se faz no modelo, nunca num aluno solto.
